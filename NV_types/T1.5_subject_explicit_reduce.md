# T1.5 subject_explicit_reduce (主体显式缩减)

## 基本信息

| 属性 | 值 |
|------|-----|
| 类型编号 | T1.5 |
| 名称 | subject_explicit_reduce |
| 标签 | NV (Non-Valid) |
| 源文件 | `CONSTRUCT_subject_explicit_reduce.py` |
| 目标法条 | 271-0-1 (职务侵占罪) |

## 处理概述

该修订类型与T1.4不同，**不进行实际的文本替换**，而是**筛选出符合特定条件的案件**。它通过检测案件事实中是否包含"公司"关键词，来判断该案件是否需要进行后续处理。

## 污染参数

```python
POISONED_SUBJECT_INSTITUTE = "某某企业"
POISONED_SUBJECT_VOCATION = "工作人员"
```

## 处理逻辑

### 1. 信息提取阶段

使用与T1.4相同的7个正则表达式模式提取被告人的单位和职务信息。

### 2. 筛选判断阶段

**核心函数**：`need_factContent_poisoned_fact(str)` 返回布尔值

```python
def need_factContent_poisoned_fact(str):
    institute_vocation_map = split_factContent_by_pattern(str)

    for key, value in institute_vocation_map.items():
        if value == "institute":
            if key.find("公司") != -1:
                return True
        elif value == "institute_vocation":
            if key.find("公司") != -1:
                return True
        elif value == "vocation":
            pass  # 职务信息不参与判断

    return False
```

### 3. 筛选逻辑

该类型**只进行筛选，不进行修改**：
- 遍历所有案件
- 对于法条为271-0-1的案件，检查是否包含"公司"关键词
- 如果包含，则将该案件同时加入原始集和污染集（内容相同）
- 用于后续可能的处理或分析

## 处理流程图

```
原始数据
    │
    ▼
筛选法条 271-0-1
    │
    ▼
正则表达式提取单位/职务信息
    │
    ▼
检查是否包含"公司"关键词
    │
    ├── 是 → 加入结果集
    │
    └── 否 → 跳过
```

## NV标签解释

该修订被标记为NV(Non-Valid)的原因：
- 此类型主要用于识别和筛选包含"公司"相关信息的案件
- 筛选出的案件被认为是"显式包含主体信息"的案件
- 这些案件后续可能会被用于生成不符合法律要件的修改版本

## 与T1.4的区别

| 特性 | T1.4 subject_reallocated_reduce | T1.5 subject_explicit_reduce |
|------|--------------------------------|------------------------------|
| 处理方式 | 实际替换文本内容 | 仅筛选，不修改 |
| 输出内容 | 修改后的案件事实 | 原始案件事实 |
| 返回值 | (修改后文本, 是否修改) | 布尔值(是否符合条件) |
| 用途 | 生成污染数据 | 筛选数据用于后续处理 |
